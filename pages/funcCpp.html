Functions in CPP

Functions in C++ are modular blocks of code that perform a specific task. They help in organizing code into manageable and reusable units. A function in C++ consists of a function declaration, a function definition, and a function call. Functions can have parameters and return values, contributing to the modular and structured design of C++ programs.

The main() function

The main function is a special function in C++ that serves as the entry point for the execution of a program. When a C++ program is run, the execution starts from the main function. The main function can call other functions, and it typically contains the statements that define the primary logic of the program. The signature of the main function is standardized as int main(), and it may also have parameters depending on the requirements of the program.

Function prototyping & Defining

Function prototyping involves declaring the function's signature (return type, function name, and parameter types) before its actual definition. This allows the compiler to recognize the function and its interface before it's used in the program. Function prototypes are often placed at the beginning of the code or in header files.

Function defining involves providing the actual implementation or body of the function. The function definition includes the code that executes when the function is called. The combination of function prototypes and definitions supports the modular structure of C++ programs, enabling the separation of interface (prototype) and implementation (definition).

In summary, functions in C++ are essential for code organization and reusability. The main function serves as the starting point for program execution. Function prototyping and defining allow for a clear separation between declaring the function's interface and providing its implementation.

In-line function

When the program executes the function call instruction the CPU stores the memory address of the instruction following the function call, copies the arguments of the function on the stack, and finally transfers control to the specified function. The CPU then executes the function code, stores the function return value in a predefined memory location/register, and returns control to the calling function. This can become overhead if the execution time of the function is less than the switching time from the caller function to called function (callee).

For functions that are large and/or perform complex tasks, the overhead of the function call is usually insignificant compared to the amount of time the function takes to run. However, for small, commonly-used functions, the time needed to make the function call is often a lot more than the time needed to actually execute the functionâ€™s code. This overhead occurs for small functions because the execution time of a small function is less than the switching time.

Example : function prototyping

        
#include <iostream >

    // Function prototype
    void displayMessage();
    
    int main() {
        // Function call
        displayMessage();
        return 0;
    }
    
    // Function definition
    void displayMessage() {
        std::cout << "This is the displayMessage function." << std::endl;
    }
